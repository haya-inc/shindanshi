## このページの役割

情報通信技術の基礎は、`IT 用語の土台` を作るページです。ハードウェア、ソフトウェア、データベース、ネットワーク、セキュリティ、Web、クラウド、AI を、`定義` と `企業での使われ方` で整理します。

## 学習のポイント

- まず `コンピュータの基礎`
- 次に `データベース`
- そのうえで `ネットワーク`
- 後半で `セキュリティ`、`Web/クラウド`、`AI`

## 最優先で固める論点

### コンピュータの基礎

CPU、メモリ、補助記憶装置、OS、ミドルウェア、処理形態は入口です。ここでは、`どの装置が何をするか`、`どのソフトウェアがどこに位置するか` をはっきり分けます。

### データベース

リレーショナル DB、主キーと外部キー、正規化、SQL、JOIN、トランザクションは最重要です。診断士試験では `SQL の基本問題` が安定して出るため、ここは確実に取れるようにします。

### 通信ネットワーク

OSI 参照モデル、TCP/IP、DNS、HTTP/HTTPS、IPアドレス、サブネット、LAN/WAN は頻出です。略語をそのまま覚えるのではなく、`どの層で何をしているか` を押さえます。

### 情報セキュリティ

CIA、暗号化、ハッシュ、デジタル署名、PKI、認証、マルウェア、ファイアウォール、VPN は定番です。ここは `守りたいもの` と `防ぎたい脅威` の対応で整理すると崩れにくくなります。

### Web とクラウド

API、Web サービス、クラウド、SaaS、PaaS、IaaS は、企業での利用場面と一緒に理解します。クラウドのサービスモデル比較は毎年押さえておきたい論点です。

## 次に固める論点

### システム構成と信頼性

負荷分散、冗長化、RASIS、稼働率、フォールトトレランスは、システムを止めない設計の基礎です。運営管理やプロジェクト管理の論点ともつながります。

### 統計解析の基礎

平均、分散、相関、回帰、検定は、後半の意思決定支援やデータ活用の基礎になります。計算式を全部覚えるより、`何を読み取る指標か` を先に押さえます。

### AI・IoT・先端技術

AI、機械学習、生成AI、IoT、ビッグデータは出題が増えています。ただし、最新ワードを増やしすぎるより、`何ができる技術か`、`どんなリスクがあるか` を整理する方が有効です。

## 2026-03-26 時点の最新確認メモ

- IPA の `情報セキュリティ10大脅威 2026` は `2026-01-29` 公開、`2026-03-12` 更新で、組織向け脅威に `AIの利用をめぐるサイバーリスク` が初選出されています。
- 総務省・経済産業省の `AI事業者ガイドライン` は `2025-03-28` 付の `第1.01版` が確認でき、AI 開発者、提供者、利用者の観点が整理されています。
- これらは更新論点ですが、試験対策としては `DB`、`ネットワーク`、`セキュリティ基礎` を落とさないことが先です。

## 典型的なつまずき

- SQL を文法暗記だけで覚えて、表の関係を追えない
- TCP/IP や DNS を、何のための仕組みか説明できない
- 暗号化、ハッシュ、デジタル署名の役割が混ざる
- SaaS、PaaS、IaaS を、利用主体ではなく語感で覚えてしまう
- 生成AIのような新語を追いすぎて、基礎論点の点を落とす

## 問題を解くときの観点

- 問われているのは `計算機の仕組み`、`データの扱い`、`通信`、`防御` のどれか
- その用語は `何をする技術か`、`どこで使う技術か`
- セキュリティなら、守る対象と対策が対応しているか
- 新しい用語でも、結局は既存の基礎論点のどこに乗るかを見極める
